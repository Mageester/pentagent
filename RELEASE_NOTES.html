What this build is trying to do
  • Choose pivots from observed evidence instead of replaying the same checklist.
  • Validate findings before promoting them into the final report.
  • Keep the operator informed with a richer live UI and better summaries.
  • Prefer Kali/Athena WSL tools when they are available on the host machine.

Release Highlights

The biggest improvements are about quality of decisions: better memory, better pivots, and less time wasted on repeated dead ends.

Core intelligence

Attack graph memory

Discovered targets, findings, and tool friction are tracked as a graph so the agent can choose the next pivot from actual evidence instead of restarting from zero each step.

Operator control

Mission-focused runs

Launch the agent with a broad objective and it will prioritize that mission first, then keep the run aligned with the most useful subgoals it discovers.

Reliability

Failure-aware tool routing

Timeouts, repeated actions, and missing tool failures now feed back into the prompt so the model can pivot instead of looping forever.

Environment aware

Kali / Athena WSL support

Network tooling prefers Kali and Athena WSL when they are present, making the agent behave more like a native pentest workstation.

Visibility

Informational console UI

The live terminal dashboard now exposes target profile, queue state, last tool, latest finding, graph depth, and tool health to make the run easier to steer.

Reporting

Readable release page

This page is the browser-friendly changelog: a clean, detailed release summary that can be shipped with the repo and opened directly in a browser.

What Changed Under the Hood

These are the code paths that most affect how the agent behaves during a real engagement.

01 / Prompting

Universal pentest directive

The system prompt now carries a consistent operating policy: prefer evidence gain over checklist repetition, keep findings provisional until validated, and treat phase order as advisory when the evidence suggests a better pivot.

02 / Reasoning

Target profiling and attack-graph guidance

The agent records whether a target behaves like a public web domain, local lab service, or network host and uses that profile to decide whether subdomain recon, content discovery, or host enumeration will produce the highest signal.

03 / Tooling

WSL-first command execution

Kali and Athena WSL are treated as first-class tool backends. That means the agent can prefer better Linux-native binaries for recon, enumeration, and exploitation support rather than falling back to weak Windows equivalents.

04 / Recovery

Timeout and repeat detection

Failed scans now feed back into the model's memory. When a tool stalls, the next action should pivot to a different technique, a narrower target slice, or a more service-specific check instead of retrying the same request.

05 / Reporting

Better terminal and browser output

The live console shows more of the internal state, the HTML report surfaces validated findings and tool health, and this release notes page gives the release itself a clear operator-facing narrative.
